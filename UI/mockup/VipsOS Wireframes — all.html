Wireframing VipsOS

Low-fi exploration before we commit. Feel over pixels. Thinking in the margins.

Designer's notes · start here

00How I'm reading this brief

The design system calls for clarity, speed, trust — for a platform that sits between an ops team who just wants to know is it broken and a platform engineer who lives in the workflow builder. Most integration platforms get one of those right and make the other feel like a punishment. I'm optimizing for both, through one shell that gracefully demotes complexity.

I'll explore three shells, three dashboards, one builder, and two storyboards. The storyboards are the acid test: if a flow can be told in four sketches, the shell is pulling its weight.

Reference vibe: n8n for the DAG builder, Retool for the shell density, Stripe for settings-surface clarity. Not copying — triangulating.

Assumptions I'm making

  • Primary personas: platform engineer (builds) + ops user (monitors). Admin and embedded are secondary for this pass.
  • Density default is spacious (Linear/Notion), with a toggle to go ops-dense.
  • Builder is strictly left-to-right DAG per your pick — not a vertical step list.
  • Everything is lo-fi intentionally. No real icons, no final type. Placeholders announce themselves.
  • Workspace multi-tenancy exists but isn't the star — shown through a workspace switcher chip, not a whole screen.

Open questions for you →

  1. Is the overview a status room, a welcome, or a checklist? I've drawn all three.
  2. Do we want workspace-scoped or global nav? Shell B implies workspace-scoped.
  3. Minimap + zoom on canvas — table stakes or nice-to-have?
  4. Run detail: single-pane with log drawer, or dedicated log page?
System constraints App shell must serve 15+ product areas. Nav can't collapse into three items — but we can tier them: primary (build) vs operate vs admin.
Principle in play "Visual hierarchy must do the work" — so I'm leaning on typographic scale + surface tint rather than color. Accent orange is reserved for action and selected.
What I'm NOT doing No final icon system. No hero illustrations. No dark-mode polish. If these wireframes survive, the hi-fi pass earns them.
Risk I'm flagging The builder canvas is dangerous to wireframe — too abstract and you design the wrong thing. I'm showing it realistic enough to pressure-test layout, but we should hi-fi it early.
